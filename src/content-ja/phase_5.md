## Phase 5: 構造化と完全性チェック

**目的**: 調査結果を整理し、全カテゴリが揃っているか確認する

### 完全性チェック表

| カテゴリ | 問い | 対応ディレクトリ |
|---------|------|-----------------|
| 理論・原則 | 「何が良いか」を知っているか？ | `foundations/` |
| 情報要件 | 「何を聞くべきか」を知っているか？ | `foundations/` |
| プロセス | 「どう進めるか」を知っているか？ | `foundations/` |
| パターン・技法 | 「どう作るか」を知っているか？ | `implementation/` |
| アンチパターン | 「何を避けるか」を知っているか？ | `implementation/` |
| 検証手法 | 「正しく作れたか」を検証できるか？ | `evaluation/` |
| 技術スタック固有 | 実際の技術で使えるか？ | `frameworks/` |
| 具体例 | 良い/悪いの判断基準は具体的か？ | `references/` |
