## Phase 3: ギャップ分析

**目的**: Phase 2 の成果物を見て「これだけでエージェントは実際に作業できるか？」を問う

Phase 1 のベースライン記録と照合し、**観測された失敗が解決されるか**を確認する。

### 判断軸

| 観点 | 問い |
|------|------|
| 理論→実践の変換 | 原則はわかったが、具体的にどう実装する？ |
| 網羅性 | 扱っていない重要なサブトピックは？ |
| 判断力 | エージェントが「良い/悪い」を自分で判定できるか？ |
| 検証可能性 | 出力が正しいことを確認する方法はあるか？ |
| 技術固有性 | 実際の技術スタックで使えるか？ |
| 具体性 | 良い/悪いの具体例があるか？ |

ギャップが見つかれば Phase 4 で埋める。
